# Guard coverage matrix

This matrix inventories the current spell-reference and client-ID surfaces. It is
organized by executable path rather than by class name so adjacent entry points do
not disappear inside a single broad verdict.

Verdicts:

- **enforced** — the application checks the guard deliberately; the named Pest
  test proves the behavior.
- **not applicable** — that path has no value governed by the guard. The pointer
  identifies the inventory boundary that makes it inapplicable.
- **GAP** — retained only as a legend value. All gaps found by this sweep were
  fixed; the “Gaps closed” section records their former locations and severity.

## Gaps closed

| Severity | Former gap | Fix and proof |
|---|---|---|
| **high** | Catalog import skipped tombstoning any referenced version, so removal could never invalidate a current selection. | Activity now changes independently of reference immutability and affected slots refresh transactionally. Proof: G3 catalog-removal test. |
| **high** | `SpellAccessBuilder` allowed inactive `kept_override` slot routes because overrides bypassed eligibility, and inactive unprepared ritual book entries because the Wizard query had no active predicate. | Both route queries require active versions. Proof: the two focused G1 access tests. |
| **medium** | New `fixed_spell` and spellbook-acquisition rules could materialize already inactive versions. | New references are rejected; exact existing character references survive regeneration. Proof: fixed-grant and spellbook-acquisition G1 tests. |
| **medium** | Level/subclass/source reconciliation orphaned choices but left `selection_eligibility = valid`. | Populated orphaned slots are marked invalid with a trigger-specific reason and are refreshed on stable-row reactivation. Proof: G3 level and subclass tests. |
| **high** | Subclass/source deactivation skipped slots already in `kept_override`, allowing a removed subclass route to remain castable. | Deactivation now reconciles both active and kept-override slots; the subclass G3 test starts from a kept override. |
| **medium** | Retained inactive Wizard spellbook content was visually indistinguishable from live catalog content. | Report data exposes activity and both UIs label inactive entries unavailable. Proof: inactive spellbook and catalog-removal tests; typecheck/build verify consumers. |
| **high** | A signed slot inverse could restore stale cached eligibility after rules changed; an inverse carrying `kept_override` could also overwrite a newer orphan state and resurrect a removed source's castable route. | Active/override restores now re-evaluate current predicates, and current orphan lifecycle wins over stale inverse state. Proof: both G3 slot-inverse restore tests. |

## Deliberate boundaries

- Existing inactive references are preserved; new inactive references are refused.
- `keep_override` may override ordinary list/level/school/tag eligibility only while
  the catalog version itself remains active. It cannot resurrect removed content.
- Snapshot/inverse restore keeps the earlier A5 contract: it may not reintroduce a
  version that became inactive after the snapshot/inverse was created.
- Spellbook/loadout ownership is not delegated to database constraints because no
  client surface exists yet. The matrix makes that future guard obligation explicit.
